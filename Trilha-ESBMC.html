ESBMC

ESBMC é um verificador de modelo limitado por contexto para software C / C ++ embutido baseado no solver Satisfiability Modulo Theories (SMT). Ele permite que o engenheiro de verificação (i) verifique o software single e multi-threaded (com variáveis ​​e bloqueios compartilhados); (ii) raciocinar sobre under- e overflow aritmético, segurança de ponteiro, vazamentos de memória, limites de array, atomicidade e violações de ordem, impasse e disputa de dados; (iii) verificar programas que fazem uso de nível de bits, ponteiros, estruturas, uniões e aritmética de ponto fixo.

O ESBMC não exige que o usuário anote os programas com pré / pós-condições, mas permite que o usuário indique propriedades adicionais usando declarações assert, que são verificadas também. Ele também fornece três abordagens (preguiçoso, gravação de agendamento e subaproximação e ampliação) para modelar o software multithread de verificação. O ESBMC pode ser chamado por meio da interface da linha de comandos ou configurado por meio do plug-in Eclipse. ESBMC converte as condições de verificação usando diferentes teorias de fundo e as passa diretamente para um solucionador SMT. Além disso, o ESBMC pode emitir condições de verificação usando as lógicas SMT QF_AUFBV e QF_AUFLIRA. ESBMC é construído em cima da estrutura CProver .

ESBMC usa os solucionadores SMT Z3 (padrão) e Boolector, que também estão incluídos na distribuição. As restrições do Z3 , Boolector e CProver licenças aplicáveis.

ESBMC é um projeto conjunto com a Universidade de Southampton , a Universidade de Stellenbosch e a Universidade Federal do Amazonas .



A Figura acima demonstra os principais componentes de software do ESBMC. As caixas brancas (exceto para o SMT solver ) representa os components que são herdados do CBMC sem nenhuma modificação enquanto que as caixas cinzas com linhas tracejadas representam os componentes que são modificados de modo a melhorar a análise inicial no GFC (Grafo de Fluxo de Controle) do programa para determinar a melhor codificação e solucionador de satisfabilidade para um particular programa e dar suporte a verificação de software multi-threaded

As caixas cinzas com linhas sólidas,na figura, representam os novos componentes que são implementados para codificar as atribuições e propriedades dadas de um programa ANSI-C para um contexto lógico global, usando os conceitos e definições das teorias suportadas pelos SMT solvers. O ESBMC também implementa um novo componente para interpretar os contra-exemplos gerados pelos SMT solvers suportados. As caixas sombreadas indicam os componentes do software que devem ser implementados no back-end para suportar cada novo SMT solver.
